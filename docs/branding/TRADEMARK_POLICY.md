# Navet Brand & Trademark Policy

This policy covers the **Navet** name, logo, wordmark, iconography, and other brand identifiers in this repository (the "Marks").

## Code vs Brand

Navet source code is licensed separately under **AGPL-3.0**.

That code license does **not** grant permission to use the Navet Marks in a way that suggests:

- official affiliation
- endorsement
- sponsorship
- an official Navet distribution

## Allowed Uses

You may:

- refer to Navet factually to describe this project or your fork
- link to the original repository
- state clearly that your project is "based on Navet"
- use unmodified screenshots for commentary, review, or documentation
- use the word "Navet" in nominative references such as compatibility notes, credits, or factual statements about project origin

## Not Allowed Without Permission

You may not:

- ship a fork under the **Navet** name
- use the Navet logo or wordmark as the branding of your modified product
- imply that your fork is the official Navet app
- use the Marks in app icons, splash screens, marketing pages, or store listings for a modified distribution
- recolor, distort, or restyle the logo in a way that creates a derivative brand mark

## Forks and Redistributed Versions

If you distribute a modified version, you may do so under the code license, but you may not brand that modified version in a way that is likely to confuse users about its origin.

In practice, modified distributions should:

- use their own product name and visual identity, or otherwise clearly avoid suggesting official Navet status
- not use the Navet logo as the product logo for the modified distribution
- make it clear when they are independent forks rather than the official Navet project

Acceptable example:

> Based on Navet, modified by Example Project.

Not acceptable example:

> Navet Pro

## Official Assets

Official brand assets remain subject to this policy even when they are stored in `public/` or elsewhere in the repository.

## Questions

If you want to use the Navet name or logo outside the allowed cases above, ask the project maintainers first.
