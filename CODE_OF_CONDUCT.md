# Code Of Conduct

## Our Commitment

We want Navet to be a respectful, technically serious, and welcoming project for users,
contributors, and maintainers.

People participating in this project are expected to:

- be respectful and professional
- assume good intent, while still being clear about technical problems
- give constructive feedback tied to the work
- avoid harassment, discrimination, personal attacks, and abusive behavior
- respect different experience levels, identities, and backgrounds

## Expected Behavior

Examples of behavior that help this project:

- asking questions in good faith
- giving clear, actionable review feedback
- acknowledging mistakes and correcting them
- keeping disagreements focused on code, behavior, documentation, and product decisions

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, intimidation, or threats
- insulting, demeaning, or inflammatory language
- discrimination based on personal characteristics
- publishing private information without permission
- repeated bad-faith disruption of discussions, issues, or pull requests

## Scope

This code of conduct applies to project spaces, including:

- GitHub issues
- pull requests
- discussions
- other communication channels used for Navet project work

## Reporting

If you experience or witness behavior that violates this code of conduct, contact the maintainers
privately at `security@navet.app`.

Include:

- what happened
- where it happened
- links or screenshots if relevant
- any immediate safety or moderation concerns

## Enforcement

Project maintainers may remove comments, close discussions, reject contributions, or block access
from people whose behavior violates this code of conduct.

Responses will be based on the severity and pattern of the behavior, with the goal of protecting
the project and its participants.
